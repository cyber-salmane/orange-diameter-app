DATABASE SCHEMA
===============

┌──────────────────┐
│ users            │
├──────────────────┤
│ id               │ UUID primary key
│ username         │ Unique, 3-30 chars
│ email            │ Unique, validated
│ password_hash    │ bcrypt hashed
│ ip               │ Last known IP
│ first_seen       │ Registration date
│ last_seen        │ Last login
│ is_banned        │ Admin flag
│ is_verified      │ Email verified
│ verification_token│ Email token
│ created_at       │ Account creation
└──────────────────┘

┌──────────────────┐

├──────────────────┤
│ id               │ UUID

│ created_at       │ Session start
│ expires_at       │ Session end (24h)
│ ip               │ Client IP
└──────────────────┘

┌──────────────────┐

├──────────────────┤

┌──────────────────┐

DATA FLOW
=========

User Registration:
------------------
Browser → app.py → auth.py → utils.py (validate) → db.py → Database
                       ↓
                  utils.py (send email) → Console

User Login:
-----------
Browser → app.py → auth.py → db.py (check credentials)
                       ↓
                  auth.py (rate limit check)
                       ↓
                  auth.py (create session) → db.py → Database
                       ↓
                  Browser (session token)

Image Processing:
-----------------
Browser → app.py → processing.py → Open3D/OpenCV/NumPy
                       ↓
                  processing.py (save results) → db.py → Database
                       ↓
                  Browser (annotated image + statistics)

Admin Access:
-------------
Browser → app.py → admin.py → db.py → Database
                       ↓
                  Browser (statistics, user lists, etc.)


SECURITY LAYERS
===============

Layer 1: Input Validation (utils.py)
    ↓ Email format, username pattern, password strength
    ↓ File type, file size, filename sanitization

Layer 2: Authentication (auth.py)
    ↓ bcrypt password hashing
    ↓ Email verification
    ↓ Rate limiting (5 attempts / 15 min)

Layer 3: Authorization (auth.py + app.py)
    ↓ Session validation
    ↓ Admin password check
    ↓ Banned user check

Layer 4: Database (db.py)
    ↓ Parameterized queries (SQL injection prevention)
    ↓ Indexed queries (performance)
    ↓ Session expiration

Layer 5: File System (processing.py)
    ↓ UUID-based filenames
    ↓ Restricted upload directory
    ↓ Automatic cleanup

PERFORMANCE CHARACTERISTICS
============================

Database Operations:
    Users table scan:        < 10ms (indexed)
    Session validation:      < 5ms (token index)
    Analysis history:        < 20ms (user_id index)
    Login attempt check:     < 10ms (username + timestamp index)

Image Processing:
    Small depth map (640x480):     ~2-5 seconds
    Medium depth map (1920x1440):  ~5-10 seconds
    Large depth map (3840x2160):   ~10-20 seconds

Concurrency:
    SQLite limitation:       ~10 concurrent writes
    Gradio server:          ~100 concurrent users
    Recommendation:         Use PostgreSQL for >50 concurrent users
